# 부담 0의 음성 입력 디자인

음성은 우리 제품의 핵심 차별화 포인트인 동시에 첫날의 가장 큰 위험 구간이다. 학습자가 마이크 켜는 순간이 두 번째 dropout 지점 (첫 번째는 Phase 3 영어 발화). 이 문서는 그 순간을 어떻게 부드럽게 통과시키는지의 설계.

## 원칙 6개

1. **첫날엔 음성을 디폴트로 강요하지 않는다.** 학습자가 선택한다.
2. **모든 음성 입력엔 텍스트 fallback이 동시에 보인다.** "말 안 되면 글로"
3. **녹음 중 시각 피드백은 있되 화려하지 않다.** 차분한 인디케이터.
4. **다시 듣기 / 다시 녹음은 무한 허용.** 라이브 부담 0.
5. **음성 분석은 학습자에게 안 보여준다.** 적어도 첫 한 달은.
6. **음성 출력 (TTS)도 학습자가 켤 때만.** 첫날 파트너는 텍스트 디폴트.

## Phase별 음성 처리

### Phase 1~2 (인사/대화 시드)
**텍스트만**. 음성 모드 선택 옵션도 안 보임. 마이크 권한 요청 안 함. 학습자가 우리 인터페이스에 익숙해질 시간.

### Phase 3 (Baseline 영어 1문장)
가장 중요한 구간. 음성 모드 진입 결정.

**파트너의 한 줄 질문**: "말로 할래요, 글로 쓸래요? 편한 쪽으로."

UI 옵션 3개:
- 🎙 **말로 할래요** (음성 모드 진입, 마이크 권한 요청 트리거)
- ⌨️ **글로 쓸래요** (텍스트 입력 박스 열림)
- 🌙 **오늘은 글로. 음성은 다음에.** (텍스트 + "음성 안내 하루 미루기" 플래그)

**중요**: 세 번째 옵션이 있어야 한다. "오늘은 글, 내일도 글로 갈 수 있다"는 신호가 음성 부담을 깎는다.

### Phase 4~5 (Direction, Commit)
다시 텍스트 디폴트. 음성 모드 학습자가 켰어도 메타 대화는 자연스럽게 텍스트로 돌아온다. (음성으로 약속 시간 등록은 어색하다.)

## 음성 모드 진입 시나리오

### 학습자가 "🎙 말로 할래요" 선택

**Step 1: 권한 요청 사전 설명 (1초)**
"마이크 한 번만 켜볼게요. 녹음한 건 내 디스크에만 남아요."

(데이터 소유 매니페스토의 첫 마이크로 적용. "내 디스크에만"이 핵심.)

**Step 2: OS 네이티브 권한 요청**

**Step 3 (a): 권한 허용**
→ 녹음 화면으로 전환

**Step 3 (b): 권한 거부**
"괜찮아요. 글로 할까요?" → 텍스트 박스로 자동 전환. 음성 안내는 1주일 봉인 (`voice_locked_until: D+7`).

**Step 3 (c): 권한 묻기 자체를 학습자가 회피 (앱 백그라운드로 보냄)**
타임아웃 30초 후 자동으로 텍스트 박스 표시. "글로 해도 돼요."

---

## 녹음 화면 UX

미니멀하다. 학원의 "지금 녹음 중!!!" 같은 빨간 점멸 X.

**보이는 것** (위에서 아래):
1. 차분한 정적 인디케이터 (작은 점, pulse 없음)
2. **음성 레벨 미터** — 12-character sparkline (Codex CLI가 쓰는 패턴 차용). 학습자가 자기 음성이 실제로 들어가는지 확인용
3. 경과 시간 (0:00 → 0:30). 색상 흐릿함. "30초 안에"를 압박 아닌 안심 신호로
4. **⏸ 일시정지** 버튼 (생각할 시간)
5. **■ 정지** 버튼

**숨겨진 것**:
- 데시벨 측정값
- 단어 수 실시간 카운트
- 발음 분석 즉시 표시

학습자에게는 "녹음 중"이라는 사실 외에 아무 정보도 안 줌. 분석은 끝나고도 안 보여줌.

---

## 정지 후 화면

녹음이 끝나면 (또는 일시정지 후 정지):

1. **▶ 재생** 버튼 — 본인 녹음을 다시 듣기
2. **↻ 다시 녹음** 버튼 — 횟수 제한 없음
3. **✓ 이걸로 보내기** 버튼

학습자가 다시 녹음을 3번 4번 해도 모델은 한 번도 압박하지 않음. 그것 자체가 학습이다 (본인 발화를 듣고 다시 시도하는 행위).

**보내기 직전 한 박자**: "확인했어요. 보낼게요." (취소 1번 가능)

## 음성 fallback 계층

다음 상황 중 하나라도 발생하면 자동 fallback:

| 상황 | 자동 처리 |
|---|---|
| 마이크 권한 거부 | 텍스트로. 음성 안내 1주일 봉인 |
| 30초 무음 (녹음 시작 후) | "지금 시끄러운가요? 글로 할래요?" |
| 환경 잡음 과다 (전사 신뢰도 낮음) | 백그라운드에 flag. 학습자에게 안 알림. 다음번에 텍스트 옵션 먼저 |
| 학습자가 녹음 도중 일시정지 후 90초 멈춤 | "글로 바꿔도 돼요. 또는 다시 녹음하셔도 되고요." |
| 학습자가 영어 대신 한국어로 말함 | 그대로 전사. "한국어로 답하셨네요. 영어로 한 번만 해볼래요? 같은 내용 그대로." (1회만 권유) |
| 전사 실패 (네트워크 등) | "잠깐 인터넷 끊겼나 봐요. 글로 한 번만 보내주실래요?" |

## 파트너의 음성 출력 (TTS)

첫날엔 파트너 응답이 **모두 텍스트**다. 이유: 학습자가 동시에 두 가지 새로운 인풋(음성 입출력) 처리하는 게 부담.

**음성 출력이 등장하는 시점**:
- Day 2 첫 알림에서 "오늘부터 내 목소리도 들려드릴까요? 끄셔도 돼요."
- Day 7 mini-mirror에서 "음성으로 들으면 더 자연스럽게 들려요. 켜볼래요?"
- 항상 학습자가 명시적 켜기

**TTS 톤 가이드**:
- 빠르지 않게 (학습자가 따라 들을 수 있도록)
- 한국어 메타 대화는 한국어 TTS (자연스러움)
- 영어 발화는 영어 TTS, 권장 voice: `marin` 또는 `cedar` (REPORT.md 참조)
- 발음 연습 모드에서는 `speed=0.85` 슬로우 옵션

**중요한 예외**: Phase 5의 "선물" (자연스러운 변형 문장)은 첫날 텍스트만. 학습자가 천천히 읽기를 원할 수 있고, TTS를 켜서 압박이 되면 안 됨. Day 2 이후 학습자가 TTS 켜면 그때부터 들려줌.

---

## 학습자가 음성 ↔ 텍스트 사이를 왔다 갔다

세션 도중 모드 전환은 한 탭으로 가능해야 함:

- 텍스트 입력 박스 옆에 항상 🎙 토글
- 음성 모드에서 입력 박스 영역 누르면 텍스트 모드
- 모델은 학습자가 어떤 모드로 답하든 동일하게 처리

이게 핵심: **모드를 자유롭게 오갈 수 있다는 사실 자체가 안전망**.

---

## 음성 데이터 저장 정책

학습자가 보낸 음성 파일은:
- `~/english-learning/_internal/audio/YYYY-MM-DD-[type].mp3`
- 학습자 디스크에만 저장 (매니페스토 5번 적용)
- 클라우드 동기화는 학습자가 명시적으로 켜야
- baseline 녹음은 별도 보존 (한 달 뒤 비교용)

학습자가 음성 파일 삭제 요청 시:
- 즉시 삭제 가능 (한 줄 명령 또는 설정)
- 전사 텍스트는 일지에 남음 (그건 학습 기록이라 학습자 자산)

## 백그라운드 음성 분석 (학습자 안 보임)

각 음성 입력당 다음을 추출 (자세한 데이터 구조는 `07-background-data.md`):

- 전사 텍스트
- 발화 시간 (초)
- 단어 수
- 한국어 섞임 비율
- 망설임 마커 횟수 ("um", "uh", "I... I" 같은 패턴)
- 끊어 말한 횟수 (1초 이상 무음 구간)
- 어휘 분석 (새 어휘, 기존 어휘)
- 문법 패턴 시도 (시제, 조동사 등)
- (옵션) 발음 신뢰도 (logprobs 가능하면) — Phase 0에서는 미사용

**학습자에게는 위 중 어떤 것도 첫 한 달 동안 보여주지 않는다.** 한 달 mirror에서 처음 일부 공개.

## 확정된 기술 전제 (Spike 결과 2026-05-26)

### 인증/구독
- **ChatGPT Plus/Pro/Team/Enterprise 구독으로 작동.** API 키 별도 불필요
- API 키만 있는 환경은 voice 사용 불가 (자동으로 text 폴백)
- **`experimentalApi` 플래그 필요** — 아직 실험 상태
- voice 사용량은 일반 Codex 한도와 **별개** 한도로 잡힘

### 양방향 실시간 음성 = Codex CLI에서 1차 작동
- `[realtime]` 블록 (`type = "conversational"`) + WebRTC 기반
- 학습자: spacebar hold-to-talk
- 에이전트: 음성으로 응답 (`gpt-realtime` 모델, 권장 voice `marin` 또는 `cedar`)
- v0.105.0 (2026-02-26) 이상 필요

### Codex 데스크탑 앱 = 텍스트 우선 + 옵션 hook TTS
- **자동 TTS 재생 공식 미지원** (Issue #21645, 2026-05-08 오픈, 미해결)
- 우회: `PostTaskComplete` 또는 PostToolUse hook으로 macOS `say` 같은 외부 TTS 명령 호출 가능
- dictation cleanup + dictation dictionary는 입력 쪽만 지원

### 발음 분석 = 부분 가능 (MVP 안)
- `gpt-4o-transcribe` + `include_logprobs` 파라미터 → word-level confidence 추출
- 한계: word-level **timestamp 없음** (sentence-level만)
- MVP 활용: mirror에서 "이번 주 자신감 낮은 단어 3개" 정도
- 음향 정밀 분석 (음소, 강세, 억양)은 MVP 밖

### MVP 음성 경험 두 갈래

**A. CLI 풀 음성 경험 (Phase 0 디폴트)**
- 실시간 양방향 음성 대화
- 데스크탑 앱보다 비개발자 친화성 떨어지지만 음성 학습으로는 가장 자연스러움
- ChatGPT 유료 구독자 한정

**B. 데스크탑 앱 텍스트 + 옵션 TTS (Phase 1 추가)**
- 학습자는 음성 dictation으로 입력, 텍스트로 받음
- hook으로 macOS `say` 명령 — 자동 TTS playback 흉내
- 모든 ChatGPT 플랜 (무료 포함) 작동

### 음성 한도 정책 (확정 2026-05-26)

**조사 결과**: 처음 우려한 "voice가 한도를 빠르게 소진한다"는 코딩 세션 기준. 영어 학습 사용 패턴은 훨씬 가벼움.

| 플랜 | 일일 Advanced Voice | 우리 사용 (5분/일) 부담 |
|---|---|---|
| Free / Go | 제한적 | voice 불가 (구독 필요) |
| Plus ($20) | 몇 시간/일 풍부 | 한도의 5~10% |
| Pro ($100/$200) | 사실상 무제한 | 무의미 |
| Business / Enterprise | 사실상 무제한 | 무의미 |

남는 불확실성:
- OpenAI가 한도를 명시적 수치로 공개 안 함 (임의 조정 가능)
- Codex realtime voice의 별도 한도는 research preview 한정 — 향후 변경 가능
- 학습자가 다른 용도(코딩, 잡담)로 voice를 많이 쓰면 우리 5분이 영향받음

**정책**: 음성을 1차 경험으로 가져가되, 가벼운 안전망을 둔다.

1. **디폴트**: voice 1차. 학습자 ChatGPT 구독으로 작동. 별도 API 키 0
2. **안전망 1 — 추적**: Stop hook이 매 세션 voice 사용량을 `progress.json`의 `voice_seconds_today`에 누적
3. **안전망 2 — 부드러운 알림**: 일일 임계치 (디폴트 30분) 도달 시 "오늘 voice 한도 슬슬 차네요. 텍스트로 마저 할래요?"
4. **하드 폴백**: ChatGPT가 "usage limit reached" 응답하면 자동 텍스트 모드 + "오늘 voice 다 썼어요. 내일 다시." 한 줄
5. **BYOK는 MVP 밖**: v2에서 power user 옵션으로 도입 검토. 매니페스토 "API 키 없이" 약속은 디폴트로 유지

근거 인용:
- ChatGPT Plus의 Advanced Voice는 "several hours daily" (사실상 풍부) — [ChatGPT Voice Mode Review 2026](https://qcall.ai/chatgpt-voice-mode-review), [Voice Mode FAQ](https://help.openai.com/en/articles/8400625-voice-mode-faq)
- Codex realtime voice는 ChatGPT 구독 인증으로 작동, voice 한도는 일반 Codex 한도와 별개 — [WebRTC Realtime Voice in Codex CLI](https://codex.danielvaughan.com/2026/04/10/webrtc-realtime-voice-codex-cli/)
