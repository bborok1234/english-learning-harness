완성 하네스의 핵심 루프

사용자가 매일 돌아오는 이유는 새 기능 때문이 아니라, 어제 말한 것이 오늘의 학습으로 돌아오기 때문이다.

1

Command

Codex에서 오늘의 학습 명령을 실행하고 learner context를 불러온다.

2

Session

Codex 대화 안에서 실제 상황 기반 3-6턴 speaking task를 진행한다.

3

Repair

막히면 rescue phrase와 prompt-first repair로 다시 말하게 만든다.

4

Store

내 삶에서 나온 문장을 로컬 파일에 저장하고 review queue에 넣는다.

5

Dashboard

HTML 표면에서 오늘의 증거, 다음 액션, 누적 여정을 확인한다.

사용자가 체감하는 최종 경험

매일 들어가고 싶어지는 이유

부담 없는 7분, 하지만 어제와 이어지는 7분

결석을 벌하지 않고, 돌아온 날을 쉽게 다시 시작한다. Codex 세션은 “오늘 뭐 하지?”가 아니라 “어제의 learner state에서 오늘 어디로 이어지는가”를 불러온다.

실제 회화 능력의 정의

유창함보다 먼저, 멈추지 않는 능력

완성 제품은 원어민처럼 말한다고 과장하지 않는다. 대신 clarification, repair, follow-up, turn-taking을 실제 상황에서 반복하게 만든다.

GenAI는 어디에 쓰이는가?

생성형 AI는 장식이나 아바타 쇼가 아니라 회화 사건을 만드는 엔진이다. 핵심은 미디어가 아니라 intention, first attempt, trouble source, scaffold, retry, transfer를 남기는 Interaction Event Graph다.

Voice first

목소리는 점수가 아니라 repair 데이터

transcript, pause, retry를 묶어 “처음엔 막혔지만 다시 말한 순간”을 저장한다. realtime은 나중에 turn-taking, interruption, backchannel 훈련으로 확장한다.

Image input

이미지는 정보격차를 만든다

사용자의 실제 사진을 “AI가 다시 그릴 수 있게 설명하기”, “빠진 정보를 묻기”, “모르는 단어를 우회하기” 과제로 바꾼다.

Image gen

이미지 생성은 scenario compiler

추상적인 커리큘럼을 구체적인 social situation으로 컴파일한다. 카드에는 일부러 모호함이나 follow-up 압력을 넣어 clarification을 유도한다.

Video later

영상은 turn-taking simulator

짧은 장면을 멈춘 뒤 즉시 반응하게 한다. 영상의 목적은 볼거리 제공이 아니라 listening-to-speaking 전환과 사회적 타이밍 훈련이다.

네 명의 핵심 학습자를 통과해야 한다

좋은 아이디어가 아니라 검증 가능한 제품이 되려면, 각 페르소나 fixture가 반복 실행되어야 한다.

지은

얼어붙는 학습자

따뜻한 시작, shame 없는 recast, “Let me think” 같은 rescue phrase가 필요하다.

민호

교정 트라우마

대화 중에는 흐름을 유지하고, 끝난 뒤 하나만 고쳐주는 계약이 필요하다.

수진

표현 욕구가 큰 학습자

교과서 문장이 아니라 자기 취향과 세계관을 영어로 말할 수 있어야 한다.

혜원

잃어버린 실력의 회복

과거와 비교해 부끄럽게 만들지 않고, 조금 더 세련된 표현 선택지를 준다.

제품 구조

최종형은 앱이 아니라 Codex-native learning runtime이다. HTML은 제품 그 자체가 아니라, Codex가 만든 학습 상태와 검증 증거를 사람이 읽기 쉽게 보여주는 관찰 표면이다.

Codex Harness

GitHub clone, one-command setup, skills, commands, hook/fallback path, Codex session context.

실행 표면
Tutor Policy

prompt-first repair, Korean re-entry, no correction dump, one-pattern mini mirror.

대화 품질
Learning Engine

scenario planner, phrase vault, review queue, learner model, weekly mirror.

학습 핵심
Evaluation

persona fixtures, seven-day simulation, rubric checks, honest dashboard evidence.

검증 장치

최종 제품은 무엇인가?

영어를 “공부했다”가 아니라 “Codex에서 오늘도 말했고, 내 표현이 로컬 기억에 쌓였고, 다음 세션에서 다시 쓸 수 있다”로 바꾸는 개인 회화 훈련 하네스다.

✓로컬 데이터와 투명한 학습 기록
✓상황 기반 말하기와 repair 훈련
✓내 삶에서 나온 표현의 저장과 재사용
✓불안한 학습자를 계속 말하게 하는 튜터 정책
✓과장 없는 검증 게이트와 성장 대시보드
다음 구현 초점

먼저 “친구가 clone해서 오늘 바로 쓰는 길”을 만든다

첫 번째 실행 목표는 화려한 앱이 아니다. Codex hook에 의존하지 않는 explicit command-wrapper fallback, 올바른 vocabulary history, scenario loop, persona fixture harness를 만들어 “매일 쓰는 학습 엔진”을 먼저 증명한다.
